# Unified Knowledge Base

Единая база знаний нужна как промежуточный слой между:

- каталогом сайта
- `Битрикс24`
- Telegram-витриной

Ключевая идея:

- все системы соединяются через `xml_id`

## Направления обмена

### Сайт -> база знаний

Из сайта берем:

- `xml_id`
- `name`
- `price`
- `old_price`
- `promo_price`
- `stock`
- `url`
- `image`
- `sku`
- `category`
- `active`

### Bitrix24 -> база знаний

Из `Битрикс24` берем:

- `popular_products`
- `promotions`
- `manager_id`
- `date_from`
- `date_to`
- `short_text`
- `promo_text`
- стадии и категории публикации

### База знаний -> Telegram

В Telegram отдаем:

- `popular_products[]`
- `promotions[]`
- готовые карточки витрины
- готовые блоки канала

## Канонический ключ

Канонический ключ системы:

- `xml_id`

Любая интеграция обязана:

- уметь находить товар сайта по `xml_id`
- уметь связывать записи `Битрикс24` по `xmlId`
- уметь собирать итоговый витринный JSON без потери связки

## Зачем это нужно

Эта база знаний позволяет:

- одинаково обращаться к данным “сверху вниз” и “снизу вверх”
- искать товар сайта по записи `Битрикс24`
- искать запись `Битрикс24` по товару сайта
- строить Telegram-витрину из одного согласованного слоя данных
