# Этап 2. Техническая архитектура

## 1. Что уже подтверждено

### Сайт

- основной боевой сервер сайта доступен
- корень проекта: `/home/bitrix/www`
- сайт работает на `Bitrix`
- реальный каталог расположен в `iblock 26`
- структура каталога уже выгружена в локальные файлы:
  - `catalog_real.json`
  - `catalog_real.js`

### Битрикс24

Webhook рабочий и отвечает.

Проверенные сущности:

- `profile`
- `scope`
- `crm.lead.fields`
- `crm.contact.fields`
- `crm.company.fields`
- `crm.deal.fields`
- `crm.product.list`
- `crm.contact.list`
- `crm.company.list`
- `crm.lead.list`
- `crm.deal.list`

Вывод:

- CRM доступна для чтения
- товары в Битрикс24 доступны через `crm.product.*`
- клиенты доступны через `crm.contact.*` и `crm.company.*`
- заявки можно строить через `crm.lead.*` или `crm.deal.*`

- категории

- наименования товаров

### Из Битрикс24 берем

- остатки

- заявки

## 3. Архитектура системы

Система состоит из 4 узлов:

1. Сайт `Счетчики Юг`
2. Telegram-канал
3. Telegram-бот
4. Битрикс24

## 4. Роли модулей

### Сайт

Источник структуры каталога:

- дерево разделов
- дерево подразделов
- карточки товаров
- контент и изображения

### Telegram-канал

Публичная витрина:

- меню канала
- переходы по разделам
- списки товаров
- прайс-посты
- посты акций
- ссылки на бота

### Telegram-бот

Интерактивный слой:

- регистрация клиента
- вход в оптовый кабинет
- показ карты клиента
- подача заявки
- запрос оптовой цены
- обращение к менеджеру

### Битрикс24

Операционный слой:

- хранение клиентов
- хранение заявок
- хранение сделок
- хранение товарных данных CRM
- остатки
- цены
- статусы клиентов

## 5. Целевая схема обмена данными

### Поток каталога

`Сайт -> Telegram-прототип / Telegram-бот / Telegram-канал`

Что передается:

- категории
- подкатегории
- бренды
- названия товаров
- изображения
- ссылки на карточки

### Поток коммерческих данных

`Битрикс24 -> Telegram-бот / Telegram-канал`

Что передается:

- розничная цена
- оптовая цена
- крупный опт
- остатки
- наличие
- статус клиента

### Поток заявок

`Telegram-бот -> Битрикс24`

Что передается:

- ФИО
- телефон
- товар / категория
- тип запроса
- комментарий
- признак оптового клиента

## 7. Предлагаемая логика MVP

### MVP-1

- подтягивание категорий и подкатегорий с сайта
- подтягивание товарных списков с сайта

### MVP-2

- подключение реальных цен из Битрикс24
- подключение остатков из Битрикс24
- экран карты клиента
- регистрация клиента в боте

### MVP-3

- оптовый кабинет
- определение статуса клиента
- персональные цены
- автопубликация обновлений в канал
