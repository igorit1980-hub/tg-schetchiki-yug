# Copy-Paste Для Настройки Bitrix24

Ниже готовый список полей для настройки `Контактов` и `Компаний` в `Битрикс24` под Telegram-карту клиента, QR-код, номер карты и оптовый режим.

```text
НАСТРОЙКА BITRIX24 ДЛЯ TELEGRAM-КЛИЕНТОВ

Сущности:
1. Контакты
2. Компании

==================================================
КОНТАКТЫ
==================================================

Создать пользовательские поля у Контакта:

1. Название: Telegram User ID
   Код: UF_CRM_TG_USER_ID
   Тип: Строка
   Обязательное: нет
   Назначение: хранение telegram_user_id

2. Название: Telegram Username
   Код: UF_CRM_TG_USERNAME
   Тип: Строка
   Обязательное: нет
   Назначение: хранение telegram_username

3. Название: Источник Telegram
   Код: UF_CRM_TG_SOURCE
   Тип: Строка или Список
   Значение по умолчанию: telegram
   Обязательное: нет

4. Название: Телефон нормализованный
   Код: UF_CRM_PHONE_NORMALIZED
   Тип: Строка
   Обязательное: нет
   Назначение: поиск и дедупликация клиента

5. Название: Тип клиента
   Код: UF_CRM_CUSTOMER_TYPE
   Тип: Список
   Значения:
   - developer
   - wholesaler
   - installer
   - retail
   - other
   Обязательное: нет

6. Название: Номер карты клиента
   Код: UF_CRM_CLIENT_CARD_ID
   Тип: Строка
   Обязательное: нет
   Назначение: ручной ввод карты
   Пример: SY-104582

7. Название: QR payload карты
   Код: UF_CRM_CLIENT_QR_PAYLOAD
   Тип: Строка
   Обязательное: нет
   Назначение: данные для QR-кода
   Пример: LOYALTY:SY-104582

8. Название: Статус карты
   Код: UF_CRM_CARD_STATUS
   Тип: Список
   Значения:
   - not_created
   - issued
   - active
   - blocked
   - archived
   Обязательное: нет

9. Название: Статус согласования
   Код: UF_CRM_APPROVAL_STATUS
   Тип: Список
   Значения:
   - new
   - pending_review
   - approved
   - rejected
   Обязательное: нет

10. Название: Разрешенный тип цены
    Код: UF_CRM_ALLOWED_PRICE_TYPE
    Тип: Список
    Значения:
    - retail
    - wholesale
    Обязательное: нет

11. Название: Скидка, %
    Код: UF_CRM_DISCOUNT_PERCENT
    Тип: Число
    Обязательное: нет

12. Название: Дата одобрения
    Код: UF_CRM_APPROVED_AT
    Тип: Дата/время
    Обязательное: нет

13. Название: Дата отклонения
    Код: UF_CRM_REJECTED_AT
    Тип: Дата/время
    Обязательное: нет

14. Название: Дата последней синхронизации
    Код: UF_CRM_LAST_SYNC_AT
    Тип: Дата/время
    Обязательное: нет

15. Название: Компания из Telegram
    Код: UF_CRM_COMPANY_NAME_SNAPSHOT
    Тип: Строка
    Обязательное: нет
    Назначение: слепок названия компании при регистрации

16. Название: Комментарий по карте
    Код: UF_CRM_CARD_COMMENT
    Тип: Текст
    Обязательное: нет

==================================================
КОМПАНИИ
==================================================

Создать пользовательские поля у Компании:

1. Название: Тип B2B компании
   Код: UF_CRM_COMPANY_TYPE_B2B
   Тип: Список
   Значения:
   - developer
   - wholesaler
   - installer
   - dealer
   - other
   Обязательное: нет

2. Название: ИНН компании
   Код: UF_CRM_COMPANY_INN
   Тип: Строка
   Обязательное: нет

3. Название: Источник синхронизации компании
   Код: UF_CRM_COMPANY_SYNC_SOURCE
   Тип: Строка
   Обязательное: нет
   Пример значения:
   - telegram
   - site
   - manual

==================================================
БИЗНЕС-ПРАВИЛА
==================================================

1. Клиент живет в Контакте.
2. Компания является B2B-контекстом клиента.
3. Карта клиента, QR и номер карты хранятся в Контакте.
4. Оптовый режим открывается только если одновременно:
   - UF_CRM_APPROVAL_STATUS = approved
   - UF_CRM_CARD_STATUS = active
   - UF_CRM_ALLOWED_PRICE_TYPE = wholesale
5. Если хотя бы одно условие не выполнено, клиент видит только розничные цены.

==================================================
ЧТО ДОЛЖНО БЫТЬ НА КАРТЕ КЛИЕНТА В TELEGRAM
==================================================

Обязательно отображать:
- ФИО
- номер карты
- QR-код
- статус карты
- тип цены
- скидку, если используется

Важно:
- номер карты нужен для ручного ввода, если сканер не работает
- QR не должен содержать открытые персональные данные

==================================================
МИНИМАЛЬНЫЙ ТЕСТ ПОСЛЕ СОЗДАНИЯ ПОЛЕЙ
==================================================

Проверить, что в API Bitrix24 появились поля:

Контакты:
- UF_CRM_TG_USER_ID
- UF_CRM_TG_USERNAME
- UF_CRM_TG_SOURCE
- UF_CRM_PHONE_NORMALIZED
- UF_CRM_CUSTOMER_TYPE
- UF_CRM_CLIENT_CARD_ID
- UF_CRM_CLIENT_QR_PAYLOAD
- UF_CRM_CARD_STATUS
- UF_CRM_APPROVAL_STATUS
- UF_CRM_ALLOWED_PRICE_TYPE
- UF_CRM_DISCOUNT_PERCENT
- UF_CRM_APPROVED_AT
- UF_CRM_REJECTED_AT
- UF_CRM_LAST_SYNC_AT
- UF_CRM_COMPANY_NAME_SNAPSHOT
- UF_CRM_CARD_COMMENT

Компании:
- UF_CRM_COMPANY_TYPE_B2B
- UF_CRM_COMPANY_INN
- UF_CRM_COMPANY_SYNC_SOURCE
```
