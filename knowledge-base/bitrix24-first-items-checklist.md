# Bitrix24 First Items Checklist

Чек-лист для первых тестовых записей, чтобы они сразу попали в `storefront.json`.

## 1. Популярный товар

Сущность:
- smart-process: `popular_products_tg`
- `entityTypeId=1042`
- `categoryId=23`
- рабочая стадия: `DT1042_23:PREPARATION` (`Активно`)

Заполнить поля:
- `title`
  Пример: `Ходовой счетчик воды`
- `xmlId`
  Должен совпадать с товаром на сайте или в `catalog_priced.json`
  Пример: `15246`
- `begindate`
  Дата начала уже наступила
  Пример: `2026-03-14T00:00:00+03:00`
- `closedate`
  Дата окончания позже текущего момента
  Пример: `2026-12-31T23:59:59+03:00`
- `sourceDescription`
  Короткий текст для Mini App
  Пример: `Проверка первой витринной записи`
- `assignedById`
  Любой ответственный пользователь

Обязательно:
- запись должна быть именно в категории `23`
- стадия должна быть именно `DT1042_23:PREPARATION`
- `xmlId` не должен быть пустым
- `xmlId` должен находиться на сайте

## 2. Акция

Сущность:
- smart-process: `telegram_promotions`
- `entityTypeId=1044`
- `categoryId=27`
- рабочая стадия: `DT1044_27:CLIENT` (`Активна`)

Заполнить поля:
- `title`
  Пример: `Тестовая акция на счетчик`
- `xmlId`
  Должен совпадать с товаром на сайте или в `catalog_priced.json`
  Пример: `577`
- `begindate`
  Пример: `2026-03-14T00:00:00+03:00`
- `closedate`
  Пример: `2026-12-31T23:59:59+03:00`
- `sourceDescription`
  Пример: `Первая тестовая акция для storefront`
- `assignedById`
  Любой ответственный пользователь

Обязательно:
- запись должна быть именно в категории `27`
- стадия должна быть именно `DT1044_27:CLIENT`
- у товара на сайте должна быть валидная пара цен:
  `old_price > promo_price`

## 3. Что проверить после создания

1. Запустить `python3 sync_backend/main.py`
2. Открыть `output/storefront.diagnostics.json`
3. Убедиться, что:
   - `category_total_count > 0`
   - `active_stage_count > 0`
4. Открыть `output/storefront.json`
5. Убедиться, что:
   - популярный товар появился в `popular_products`
   - акция появилась в `promotions`

## 4. Если запись не попала

Проверь по порядку:
- webhook читает нужный smart-process
- запись создана в правильной категории
- запись переведена в правильную стадию
- `xmlId` есть и совпадает с сайтом
- `begindate` и `closedate` покрывают текущее время
- для акции у сайта есть корректные `promo_price` и `old_price`
